# PRD: キーボードぼうけん

**バージョン**: 1.0.0
**作成日**: 2026-04-24
**ステータス**: 正式版

---

## 1. 概要

未就学児〜小学生低学年のキーボード未経験の子供が、RPGゲームとして遊ぶうちにブラインドタッチを自然に習得することを目的としたWebアプリ。棒人間キャラクターを操作して敵と戦い、お金を貯めて装備を整えながら世界を冒険する。

---

## 2. ターゲットユーザー

| 項目 | 内容 |
|------|------|
| 主な利用者 | 未就学児〜小学生低学年（4〜8歳程度） |
| キーボード経験 | 未経験〜初心者 |
| 利用シーン | 自宅での自習、保護者の見守りのもとでの練習 |
| 利用端末 | PC（物理キーボード必須） |
| 言語 | 日本語のみ |

---

## 3. ゴール

### 学習ゴール
- アルファベット26文字のローマ字入力をブラインドタッチで打てるようになる
- 最終的にひらがなをローマ字入力で打てるようになる（例：「ねこ」→ `neko`）

### プロダクトゴール
- 「気づいたらブラインドタッチができていた」という体験を実現する
- 1セッション 10〜15分 で区切れる適切なテンポ設計
- アカウント不要で始められ、続きはブラウザに自動保存される

---

## 4. 機能要件

### 4.1 学習システム：ステージ全体構成

全46ステージ構成。フェーズA→B→C→D→E→F の順に進み、スキップ不可。
詳細は `.claude/rules/typing-progression.md` を参照。

#### フェーズA: アルファベット位置学習（Stage 1〜12）

ホームポジションを起点に、指の動きが最小なキーから順に解放する。

| Stage | 解放キー | 習得フォーカス |
|-------|---------|--------------|
| 1 | `J` `F` | 両人差し指・ホームポジション認識 |
| 2 | + `K` `D` | 右中指・左中指 |
| 3 | + `L` `S` | 右薬指・左薬指 |
| 4 | + `A` `;` | 右小指・左小指（ホーム行完成） |
| 5 | + `G` `H` | 人差し指の伸ばし |
| 6 | + `E` `I` | 上段・中指 |
| 7 | + `R` `U` | 上段・人差し指 |
| 8 | + `T` `Y` | 上段・人差し指伸ばし |
| 9 | + `W` `O` | 上段・薬指 |
| 10 | + `Q` `P` | 上段・小指 |
| 11 | + `Z` `X` `C` `V` | 下段・左手 |
| 12 | + `B` `N` `M` `,` `.` | 下段・右手、アルファベット完成 |

### 4.2 出題形式：フェーズ別表示

| フェーズ | 出題内容 | 表示形式 |
|---------|---------|---------|
| A | アルファベット1〜3文字 | 大きな文字列（例：`f`、`fjk`） |
| B〜B'' | ひらがな1文字 | 大きなひらがな＋薄いローマ字ヒント（例：「し」下に薄く `shi`） |
| C〜F | ひらがな単語 | ひらがな単語＋薄いローマ字ヒント（例：「ねこ」下に薄く `neko`） |

- **攻撃は常に1キー入力単位**。1文字のローマ字が複数キー（`shi`=3打）でも1キー押すごとに攻撃が発生する
- ヒントの「薄いローマ字」は表示・非表示をユーザーが切り替え可能（キーボードガイドと同じ扱い）
- `「ん」` は単独出題しない。フェーズC以降の単語中にのみ登場する

### 4.2.1 ローマ字入力ルール

初学者の混乱を防ぐため、各文字の入力方法は一通りのみとする。

| ひらがな | 入力 | ひらがな | 入力 |
|---------|------|---------|------|
| ち | `chi` | し | `shi` |
| つ | `tsu` | じ | `ji` |
| ふ | `fu` | ん | `n`（次の文字が `na/ni/nu/ne/no` の場合は `nn`） |
| 拗音（しゃ等） | `sha` / `chi` + `ya` 等、最短形のみ | 促音（っ） | 次の子音を重ねる（例：`kitte` → `k-i-t-t-e`） |

### 4.3 バトルシステム

#### 基本フロー

```
① 敵が登場 → 出題文字が表示される
② プレイヤーがキーを押す
    ├── 正解 → ヒーローが攻撃 → 敵のHPが減る → 次の文字へ
    └── 不正解 → 正しいキーを押すまで先に進めない（ペナルティなし）
③ 敵HPが0 → 敵を撃破 → コイン獲得 → 次の敵へ
④ 規定数の敵を撃破 → ステージクリア
```

#### キャラクター仕様

| 項目 | 内容 |
|------|------|
| 主人公 | 棒人間（シンプルな線画） |
| 敵 | ステージに応じた敵キャラ（1体ずつ）、棒人間ベース or シンプル図形 |
| 同時出現数 | 敵は常に1体 |

#### HP・ダメージ数値

| ステージグループ | 敵HP |
|----------------|------|
| Stage 1〜2 | 3 |
| Stage 3〜6 | 5 |
| Stage 7〜12 | 8 |
| Stage 13〜 | 10 |

| 武器 | 攻撃力（1キー入力あたりダメージ） |
|------|-------------------------------|
| 木の剣（初期装備） | 1 |
| 鉄の剣 | 2 |
| 炎の剣 | 3 |

#### ダメージ・演出

| タイミング | 演出 |
|-----------|------|
| 正解キー入力 | ヒーローが攻撃アニメーション → 敵がダメージフラッシュ → ダメージ数値が浮き上がる |
| 敵HP0 | 敵が倒れるアニメーション → コイン獲得エフェクト → 次の敵登場 |
| ミスキー入力 | 入力欄が赤くなる（ミス表示）、敵の反撃はなし |

### 4.4 レベルアップ・ステージ進行条件

ステージクリアには**両方の条件**を満たす必要がある。

| 条件 | 内容 |
|------|------|
| 条件① 撃破数 | 規定数の敵を倒す（下表参照） |
| 条件② 正確さ | そのステージの入力精度が**70%以上** |

#### ステージごとの撃破数

| ステージグループ | クリア必要撃破数 |
|----------------|---------------|
| Stage 1〜2 | 5体 |
| Stage 3〜6 | 8体 |
| Stage 7〜12 | 10体 |
| Stage 13〜 | 10体 |

- 精度が足りない場合、クリア後に「もう一度練習しよう！」と表示し再挑戦を促す
- 精度 = 正解キー入力数 ÷ 総キー入力数（ミス含む）

### 4.5 経済システム（コイン・ショップ）

#### コイン獲得

| 要素 | 内容 |
|------|------|
| 基本報酬 | 敵の強さ（ステージ深度）に応じた基本コイン |
| ボーナス | 入力精度が高いほど獲得コインが増える（精度100%で最大ボーナス） |
| 計算式 | 基本コイン × (0.5 + 精度 × 0.5) |
| 防御装備ボーナス | 装備中の防御装備に応じてコイン獲得量がさらに増加（下表参照） |

#### ショップ

- ワールドマップ画面からいつでもアクセス可能
- 購入したアイテムは即時反映

| カテゴリ | アイテム | 効果 |
|---------|---------|------|
| 攻撃装備 | 木の剣（初期） | 攻撃力 1 |
| 攻撃装備 | 鉄の剣 | 攻撃力 2 |
| 攻撃装備 | 炎の剣 | 攻撃力 3 |
| 防御装備 | ぼうし | コイン獲得ボーナス +10% |
| 防御装備 | かぶと | コイン獲得ボーナス +20% |
| 防御装備 | マント | コイン獲得ボーナス +30% |
| アクセサリ | お守り、ブーツ | 特殊効果（将来拡張） |

- 棒人間の見た目に装備品が反映される（剣を持つ、帽子をかぶるなどシンプルな差分）
- 同カテゴリは1つだけ装備可能（上位装備に買い替える方式）

### 4.7 単語・文章リスト（Stage 13〜）

実データは `/data/words.json` で管理し、以下の基準・難易度分類に従って選定する。

**選定基準：**
- 小学1〜2年生の国語教科書レベルの語彙
- ひらがなのみ（カタカナ・漢字を含まない）
- 子供が知っている具体的なもの（動物・食べ物・自然・日常動作）

**難易度分類：**

| 難易度 | 含む音 | 例 |
|-------|-------|----|
| やさしい | 直音のみ | ねこ、いぬ、はな、そら、くも |
| ふつう | 長音・促音（っ） | おかあさん、きって、おおかみ |
| むずかしい | 拗音（しゃ・ちゅ・きょ等） | きょうりゅう、しゃしん、ちょうちょ |

### 4.8 演出・フィードバック

#### サウンド

| タイミング | サウンド |
|-----------|---------|
| 正解キー入力 | 打鍵音・爽快な効果音 |
| 攻撃ヒット | 攻撃音 |
| 敵撃破 | 撃破音・コイン音 |
| ステージクリア | ファンファーレ |
| ミス入力 | 短いブザー音 |
| アイテム購入 | ショップ効果音 |

#### BGM

| シーン | 内容 |
|-------|------|
| ワールドマップ | 明るく冒険感のあるBGM（ループ） |
| バトル中 | テンポよくリズミカルなBGM（ループ） |
| ステージクリア | 短いファンファーレ（効果音と兼用） |

- Web Audio API または `<audio>` タグで実装
- UIにミュートボタンを設置（保護者・学校環境への配慮）
- 音源はPixabay・魔王魂等の商用利用可・著作権フリー素材を使用
- BGMファイルは `/assets/sounds/` に配置

#### キーボードガイド表示

画面下部に手SVG＋キーボード図を表示する。詳細仕様は `.claude/rules/finger-guide.md` を参照。

**手SVG（指使いガイド）:**
- キーボードの上に左手・右手のSVGを横並びで表示
- 指ごとに色分け（人差し指=黄・中指=緑・薬指=青・小指=紫）で統一
- 次に押すべき指だけ明るくハイライト、他の指はグレーアウト
- 人差し指の先端に●を常時表示してF・Jキーのホームポジション突起を表現
- 正解入力時: 該当指が押し込みアニメーション（0.15秒）
- ミス入力時: 手SVG全体が赤くフラッシュ（キーボードと同期）

**キーボード図:**
- 各キーを指に対応した色でハイライト
- F・J キーに下線マークを常時表示（ホームポジション識別）
- 現在のステージで使えるキーのみ色付き表示、他はグレーアウト
- **表示・非表示はプレイヤーが自由に切り替え可能**（手SVGとキーボード図は連動して切り替わる）

**初回ホームポジションガイド:**
- フェーズA Stage 1 の初回プレイ時のみ、手SVGの上に吹き出しを表示
- 最初のキー入力で自動的に消える（以降は表示しない）

## 5. 非機能要件

### 5.1 技術スタック

| 項目 | 内容 |
|------|------|
| フロントエンド | HTML / CSS / JavaScript（フレームワークなし） |
| ホスティング | GitHub Pages |
| 外部依存 | なし（オフラインでも動作） |

### 5.2 対応ブラウザ

| ブラウザ | 対応 |
|---------|------|
| Chrome（PC） | ✅ |
| Edge（PC） | ✅ |
| Firefox（PC） | ❌ |
| Safari（Mac） | ❌ |
| スマホ・タブレット | ❌（物理キーボード必須のため対象外） |

### 5.3 データ管理

| データ | 管理方法 |
|-------|---------|
| ステージ定義・敵データ | 外部JSONファイルで管理（`/data/stages.json` 等） |
| 単語・文章リスト | 外部JSONファイルで管理（`/data/words.json` 等） |
| ショップアイテム定義 | 外部JSONファイルで管理（`/data/items.json` 等） |

- JSONファイルを差し替えるだけでコンテンツを追加・修正できる設計

### 5.4 セーブ機能

- **ブラウザのlocalStorageに自動保存**（アカウント不要）
- 保存タイミング：ステージクリア時、アイテム購入時
- 保存データ：現在ステージ、所持コイン、装備状態、各ステージの最高精度
