# キーボードぼうけん — 画面デザイン指示書

Claude Design へのプロトタイプ作成依頼ドキュメント。

---

## 全体デザイン方針

### コンセプト
- **対象**: 4〜8歳の子供
- **雰囲気**: 明るく楽しいRPG。難しそうに見えない、シンプルで直感的なUI
- **キャラクター**: 棒人間（シンプルな線画）を基本とする
- **文字**: ひらがな中心。英数字は大きく見やすく

### カラーパレット
- プライマリ: 明るい黄色 `#FFD700`（コイン・強調）
- セカンダリ: 空色 `#5BB8FF`（ボタン・ハイライト）
- 成功: 緑 `#4CAF50`
- ミス: 赤 `#FF5252`
- 背景: クリーム `#FFF9E6`（目に優しい）
- テキスト: 濃いネイビー `#1A237E`

### フォント方針
- 日本語: 丸みのあるゴシック体（例：Noto Sans JP）
- キー表示: モノスペース・太字
- サイズは全体的に大きめ（子供が読みやすい）

### レイアウト基本
- 画面サイズ: 1280×720px（PC横向き固定）
- スマホ非対応（物理キーボード必須のため）

## 画面詳細

### 1. タイトル画面

**目的**: ゲーム開始・続きから再開

**レイアウト:**
```
┌─────────────────────────────────────┐
│  （空・雲などの背景イラスト）             │
│                                     │
│       ⚔️ キーボードぼうけん ⚔️          │
│        （タイトルロゴ・大きく）           │
│                                     │
│        [はじめる]  [つづきから]         │
│          （大きいボタン2つ）             │
│                                     │
│      棒人間キャラクターが剣を持って立つ    │
└─────────────────────────────────────┘
```

**UI要素:**
- タイトルロゴ: 中央上部、影付き・太字
- 「はじめる」ボタン: 黄色背景・大きめ（子供が押しやすい）
- 「つづきから」ボタン: 空色背景（セーブデータがない場合はグレーアウト）
- 棒人間: 画面下部中央、冒険者のポーズ（剣を掲げている）
- 背景: 明るい草原・空のイラスト

### 2. ワールドマップ画面

**目的**: ワールド・ステージ選択、ショップ・装備確認

**レイアウト:**
```
┌─────────────────────────────────────┐
│ [👤装備] コイン: 💰 xxx   [🛒ショップ] │  ← ヘッダー
├─────────────────────────────────────┤
│                                     │
│  ●━━━●━━━●    ●━━━●━━━●          │
│ S1  S2  S3  S4  S5  S6             │
│ （クリア済みは金色、現在は点滅、未解放はグレー）│
│                                     │
│  ワールド名: もりのみち                 │
│                                     │
└─────────────────────────────────────┘
```

**UI要素:**
- ヘッダー左: 主人公の棒人間（装備を反映した見た目）＋装備確認ボタン
- ヘッダー右: 所持コイン数（コインアイコン付き）、ショップボタン
- マップエリア: ステージがパスで繋がった横スクロールマップ
  - クリア済みステージ: 金色の丸アイコン＋チェックマーク
  - 現在挑戦可能ステージ: 点滅する明るい丸アイコン
  - 未解放ステージ: グレーの鍵アイコン
- ワールド区切り: ワールド名のラベル
- ステージを選択するとポップアップが出て「挑戦する！」ボタン

### 3. バトル画面（メイン）

**目的**: キーボード入力練習のメインゲームプレイ

**レイアウト:**
```
┌─────────────────────────────────────┐
│ Stage 3  精度: 82%  撃破: 3/8       │  ← ステータスバー
├───────────────────┬─────────────────┤
│                   │                 │
│   【主人公エリア】   │  【敵エリア】      │
│                   │                 │
│   棒人間（装備付き）  │  敵キャラ         │
│      ⚔️→          │  HP ████░░░      │
│                   │  （HPバー）        │
│                   │                 │
├───────────────────┴─────────────────┤
│                                     │
│         【出題エリア】                 │
│                                     │
│    ねこ          ↑現在入力中の文字     │
│    neko          ↑薄いローマ字ヒント   │
│    [n][e][ ][ ]  ↑入力進捗インジケータ  │
│                                     │
├─────────────────────────────────────┤
│  【キーボードガイド】  [ガイドを隠す]    │
│  Q W E R T Y U I O P               │
│   A S D F G H J K L ;              │
│    Z X C V B N M , .               │
│  （押すべきキーが青くハイライト）          │
└─────────────────────────────────────┘
```

**UI要素:**

*ステータスバー（上部）:*
- 現在ステージ番号
- リアルタイム精度（%）
- 撃破数カウンター（現在/目標）

*主人公エリア（左半分）:*
- 棒人間キャラクター（装備を視覚的に反映）
- 攻撃時: 剣を振るアニメーション（右に向かって）
- 待機時: 軽くアイドルアニメーション

*敵エリア（右半分）:*
- 敵キャラクター（ワールドに対応したシンプル図形）
- HPバー（上に表示、減るごとに赤くなる）
- ダメージ時: 赤くフラッシュ＋ダメージ数値が上に浮き上がる
- 撃破時: 倒れるアニメーション＋コインエフェクト

*出題エリア（中央）:*
- お題テキスト: 大きくはっきりと（Stage 1〜12はアルファベット1〜3文字、Stage 13〜はひらがな）
- ローマ字ヒント: お題の下に薄いグレーで表示（切り替え可能）
- 入力進捗インジケータ: 1文字ずつ□が並び、入力済みが青く塗られる
- ミス時: 入力欄全体が赤くフラッシュ

*キーボードガイド（下部）:*
- QWERTY配列のキーボード図
- 現在のステージで使えるキー: 空色でハイライト
- 今押すべきキー: 黄色で強調ハイライト
- 未解放キー: グレーアウト
- 「ガイドを隠す/表示する」トグルボタン（右端）

### 4. ステージクリア画面

**目的**: クリア結果表示・次ステージへの誘導

**レイアウト:**
```
┌─────────────────────────────────────┐
│                                     │
│        🎉 ステージクリア！ 🎉          │
│                                     │
│   棒人間がガッツポーズ                 │
│                                     │
│   せいかくさ:  82%  ★★☆             │
│   たおした数:  8/8                   │
│   もらったコイン: 💰 +45              │
│                                     │
│   [もう一度]        [つぎへ →]        │
│                                     │
└─────────────────────────────────────┘
```

**バリエーション（精度不足でクリアできない場合）:**
```
┌─────────────────────────────────────┐
│                                     │
│      もうすこし！ またれんしゅうしよう！   │
│                                     │
│   棒人間が頭を掻いているポーズ           │
│                                     │
│   せいかくさ:  65%  （70%以上でクリア）  │
│   たおした数:  8/8                   │
│                                     │
│          [もう一度やる！]              │
│                                     │
└─────────────────────────────────────┘
```

**UI要素:**
- クリア時: 紙吹雪エフェクト、ファンファーレ演出
- 星評価: 精度に応じて1〜3星（70%=1星、85%=2星、95%=3星）
- 獲得コイン: アニメーションで数字が増えていく
- 「つぎへ」ボタン: 大きく・目立つ（プライマリカラー）
- 「もう一度」ボタン: 小さめ・サブカラー

### 5. ショップ画面

**目的**: 装備購入・アイテム確認

**レイアウト:**
```
┌─────────────────────────────────────┐
│ 💰 所持コイン: 380           [閉じる] │
├────────────┬────────────────────────┤
│            │  ⚔️ 攻撃装備             │
│  棒人間     │  木の剣  攻撃+1  [装備中] │
│ （現在装備  │  鉄の剣  攻撃+2  [100💰] │
│  を反映）   │  炎の剣  攻撃+3  [300💰] │
│            │                        │
│            │  🛡️ 防御装備             │
│            │  ぼうし  コイン+10% [装備中]│
│            │  かぶと  コイン+20% [80💰] │
│            │  マント  コイン+30% [250💰]│
└────────────┴────────────────────────┘
```

**UI要素:**
- 左パネル: 主人公の棒人間（現在の装備を視覚的に反映したプレビュー）
- 右パネル: カテゴリ別アイテムリスト
  - 購入済み＆装備中: 「装備中」バッジ（緑）
  - 購入済み・未装備: 「装備する」ボタン
  - 未購入（購入可能）: コスト表示＋「かう」ボタン（黄色）
  - 未購入（コイン不足）: グレーアウト＋鍵アイコン
- 購入時: コインが減るアニメーション＋装備が棒人間に反映される演出

## アニメーション・演出仕様

| 演出 | 内容 |
|------|------|
| 攻撃 | 主人公が右に0.2秒スライド → 元の位置に戻る |
| ダメージ | 敵が0.1秒赤くフラッシュ × 2回 |
| ダメージ数値 | 「-3」などの数字が敵の上に浮かんで消える（0.5秒） |
| 撃破 | 敵が下に倒れる（0.3秒）→ コインが弾けるエフェクト |
| ミス入力 | 入力欄が0.3秒赤くなる |
| クリア | 紙吹雪が画面上から降る |
| コイン獲得 | 数字がカウントアップするアニメーション |

## 棒人間デザイン仕様

### 基本形
- 頭: 円
- 体: 縦線
- 腕: 2本の斜め線
- 足: 2本の斜め線
- 線の太さ: 3〜4px、濃いネイビー

### 装備の差分表現（シンプルな追加パーツ）

| 装備 | 見た目変更 |
|------|---------|
| 木の剣 | 右手に棒状の剣 |
| 鉄の剣 | 右手に少し太い剣（銀色） |
| 炎の剣 | 右手に剣＋炎エフェクト（オレンジ） |
| ぼうし | 頭の上に三角帽子 |
| かぶと | 頭を覆うU字型 |
| マント | 背中から左右に広がる台形 |

### 敵キャラクターデザイン例

| 敵名 | 形状 |
|-----|------|
| スライム | 丸い雫型・水色 |
| キノコ | 傘型（半円＋棒）・赤白 |
| 小鬼 | 棒人間＋小さい角2本・緑 |
| コウモリ | 楕円体＋三角の翼2枚・黒 |
| ガイコツ | 白い棒人間＋丸い頭 |
| 石ゴーレム | 四角いブロック積み・グレー |

## プロトタイプ作成の優先順位

1. **バトル画面**（最もプレイ時間が長い・コアUX）
2. **ワールドマップ画面**（ナビゲーションの中心）
3. **ステージクリア画面**（達成感の演出）
4. **ショップ画面**（成長実感）
5. **タイトル画面**（導線のみシンプルでよい）
